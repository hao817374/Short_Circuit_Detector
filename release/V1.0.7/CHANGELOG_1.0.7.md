# Short Circuit Detector v1.0.7 版本说明

## 缺陷修复

- **方向校准矩阵计算失效**:
    - 修复 CALIBRATION 页面"确认并应用"按钮未调用 `finishCalibration()` 的致命 Bug。
    - 此前完成 NW/SW 采样后点击确认仅切换视图，校准矩阵保持恒等变换，方向校准完全无效。
    - 修复后确认按钮在已校准状态下先计算仿射变换矩阵再跳转罗盘页面。

- **罗盘方向映射全线修正**:
    - `DEFAULT_MAP` 坐标系转换公式最终确定为 `heading=(360-ref)%360`，将 atan2 数学角度（0°=右/东，逆时针）正确转换为罗盘地理角度（0°=上/北，顺时针）。
    - 同步修正 `Compass.tsx` 方向文字映射为与 DEFAULT_MAP 一致的地理顺时针排列。
    - 8 个方向全部验证通过：右上角激励源→罗盘指向左下方（指向 PCB 中心），右下角激励源→罗盘指向左上方（指向 PCB 中心）。

- **校准弹窗 NW/SW 按钮标签颠倒**:
    - 修复全屏校准弹窗（`isCalibrating2P`）中 NW/SW 按钮的中文标签与变量 key 映射颠倒的 Bug。
    - 此前弹窗中 NW 按钮显示"右下角激励源"、SW 按钮显示"右上角激励源"，导致 `finishCalibration()` 将采集数据分配到错误的目标矢量。
    - 主校准页面（CALIBRATION 视图）和 DebugChart 中的标签映射均无此问题。

- **DebugChart ANGLE 圆盘指针视觉偏差**:
    - 修复 ANGLE 圆盘（原始 atan2 角度）指针旋转公式：`rotate(-rawAngle)` → `rotate(90-rawAngle)`。
    - 此前指针从上方起始、用逆时针旋转，导致 atan2=0°（右/东）时指针指向上方，视觉偏差 90°。

## 新增机制

- **校准矩阵版本迁移** (`CALIB_MATRIX_VERSION`):
    - 引入版本号机制，升级时自动检测旧版本存储的校准矩阵并清零。
    - 避免方向映射公式变更后旧校准数据导致错误指向。

## 代码注释

- 为核心算法模块添加中文注释，覆盖 DSP 修剪均值、commitFrame 数据管线、零点校准、方向校准矩阵求解（含 Cramer 法则推导）、串口读取循环、端口配置、坐标系转换等关键代码段。
